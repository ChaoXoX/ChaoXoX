# AI 路线图

## 阶段 1：感知与基线
- 选模型：先用开源基线（例如 Llama 3 / Qwen）做 POC。
- 数据收集：分类场景、整理对话/任务样例、建立数据字典。
- 评价基线：人工打分 + 自动指标（准确率、覆盖率、响应延迟）。

## 阶段 2：检索与记忆
- 构建 RAG：清洗、分块、索引、召回、多路排序。
- 缓存与记忆：对高频问答、热点数据做缓存；短期记忆用对话上下文压缩。
- 监控：召回率、相关性、幻觉率（基于引用校验）。

## 阶段 3：Agent 与流程
- 能力拆分：理解、计划、工具调用、执行、总结。
- Workflow：将 Agent 动作放入可观测的 DAG，易于重试与 A/B。
- 安全与合规：输入过滤、输出约束、操作审计。

## 阶段 4：微调与迭代
- 数据：高质量指令 + 真实反馈；偏好对齐。
- 训练：LoRA/QLoRA，监控 loss 与 overfit；多轮对话需特殊模板。
- 评测：自动指标 + 人审；对比未微调模型是否显著提升。

## 阶段 5：上线与运维
- SLA：响应延迟、错误率、成本上限。
- 观测：日志、链路追踪、提示与模型版本管理。
- 灰度：按用户、按场景分流；快速回滚到安全配置。
